# 顺子技巧

## 概述

顺子是掼蛋一种常见而又具有战略意义的牌型。顺子相当于兵器库中的诸葛连弩，打出去的速度很快且难于防守。同时，顺子弊端在于牺牲其他潜在的牌型，拆掉手上的对子 (`Pair`)、三带 (`ThreeWithTwo`) 甚至5头炸 (`Bomb`) 等，从而在牌力分布上可能呈现出弱势。

**平台变量名**：
- `Straight`：顺子
- `Pair`：对子
- `ThreeWithTwo`：三带二
- `Bomb`：炸弹

## 适用阶段

- **开局阶段**：✅ 适用，可以首发
- **中局阶段**：✅ 主要适用阶段
- **残局阶段**：✅ 完全适用，残局顺子是关键

  - 钢板 (`TwoTrips`)：0.6手

## 一、顺子的定义特点

**平台变量名**：`Straight`（顺子）

**定义**：顺子是指五张连续单牌组成的牌型。顺子的牌数较为特殊，能够迅速改变局势。

**特点**：

1. **牌型相对比较固定，张数比较多**：也是许多牌手一开始组牌时经常会优先考虑选用的一种牌型

2. **顺子缺乏变牌的灵活性**：
   - 特别是在打出一两手顺子后，余牌很难再组合成其它牌型来使用
   - 勉强组成一两手顺子，余下比较多的单牌 (`Single`)，增多手数，跟牌套牌会比较困难

3. **顺子有效减少手数与小单牌的出手难度**：会使我方手中的牌打得更加流畅

4. **在单张控牌实力不足，特别是单张较多又没有大王 (`R`)、小王 (`B`)**：优先考虑组成顺子的牌型

5. **顺子的5、10法则**：
   - 所有的小顺子（如A2345到56789）都必定包含5
   - 所有的大顺子（如678910到10JQKA）都必定包含10
   - 牌局中所有的5或10都打出后，判断在不使用红心级牌的情况下，不会再出现小顺子或大顺子
   - 有助于评估手上尚未打出的顺子牌是否能冲刺成功或引到对手的炸牌

6. **组顺难度不高，调整大小的代价也比较低**：另外顺子最大也就费个A，所以俗话说"顺子封到顶"

**平台变量名**：
- `Single`：单张
- `R`：大王
- `B`：小王

## 二、顺子组牌

### 1. 顺子的组牌内单牌不应超过两张

**核心原理**：
1. **组顺生两单，肯定没眼光**：顺子的战略价值在于帮助玩家快速减少手牌数量，而非形成压制对手的牌力
2. **单张特强，双吃贡，进贡进的级牌，留对子也是拆开出**：可拆单张组顺子
3. **管压住别人的顺子，特别是助攻**：不惜拆任何牌管

### 2. 根据还贡和首发时间判断是否有顺子

**判断方法**：
- 迟迟不还贡，顺牌在蠢动
- 出牌不走牌，顺子马上来
- 中途不出牌，还在理顺牌
- 还贡牌在6到10点位，越大确认有顺，恐有同花顺 (`StraightFlush`)

**平台变量名**：`StraightFlush`（同花顺）

### 3. 拆炸组顺子

**策略**：
1. **多单成顺可拆一炸**：如组小顺23456，多出555+44，带走三单
2. **不宜破两炸组大同花顺 (`StraightFlush`)**：除非助攻阻对手方
3. **拆炸弹 (`Bomb`) 一般拆5头的变成4头**：轻易不要把4头炸拆成顺子加3头

**平台变量名**：
- `StraightFlush`：同花顺
- `Bomb`：炸弹

### 4. 红花配后出

**核心原理**：己有多把顺 (`Straight`)，只有一红配，其中两把顺，中顺和小顺，都能配同花 (`StraightFlush`)，机动关键张，后出这一张，红配配啥看情况。

### 5. 还贡分清对方和对家

**案例**：如1010+99+88+77+66+5，对手方进贡，还10（注意花色），组两把顺；对家进贡，还单5，可组三连对 (`ThreePair`) 或顺牌。

**平台变量名**：`ThreePair`（三连对）

### 6. 对手方要顺就弃顺

**核心原理**：234445556可以组小顺23456，多出44+55，发现对方要顺 (`Straight`)，自己是助攻即刻弃顺，还三单3、6，2为主。

### 7. 发现对方有顺，及时组大顺

**核心原理**：最后两三手，留大顺 (`Straight`) 候对手，发现对方开始拆小顺，己要跟拆大顺。

## 三、顺子出牌技巧

### 1. 牌弱先出顺，牌强后出顺

**核心原理**：
- 牌较弱，考虑先出顺子 (`Straight`)，以减手牌数量
- 牌强只有一手小顺子是不宜先出，出去容易被对手接手，等到接近残局阶段，看牌局情况再决定什么时候打出

### 2. 小顺往前凑，大顺必殿后

**核心原理**：
- 牌中有多个顺子 (`Straight`)，先出较小的顺子
- 小顺子用于牵制对手，保留大顺子以应对后续的牌局变化
- 留意对手的出牌反应，以便适时使用大顺子反击
- 只有一个顺子，可以考虑在关键时刻出牌，以改变场上的局势

### 3. 顺子多了有点累，单张多了活受罪

**核心原理**：
- 顺子虽好，却不能贪多
- 顺子的压制力相对较低，没有足够的炸弹 (`Bomb`) 或大牌支持，顺子往往难以在关键时刻夺取或保持出牌权
- 组顺后手中单张 (`Single`) 过多，需要高牌来收回牌权，极大地消耗了牌力

**平台变量名**：
- `Bomb`：炸弹
- `Single`：单张

### 4. 谁打谁收

**核心原理**：
- **先顺后夯还有顺，先夯后顺还有夯**
- 先出顺子 (`Straight`)，至少有9-K顺能收，对手不要后出夯 (`ThreeWithTwo`)，说明还有顺子，对家可送顺子
- 先出夯 (`ThreeWithTwo`)，至少有KKK能收，对手不要后出顺，说明还有夯，对家可送夯

**平台变量名**：
- `Straight`：顺子
- `ThreeWithTwo`：三带二（夯）

### 5. 后发制对手

**核心原理**：对手方出顺 (`Straight`)，也有两把顺，都比对手方小，等对方顺打完，炸 (`Bomb`) 后反出顺。

**平台变量名**：`Bomb`（炸弹）

### 6. 顺子中间牌的打法

**策略**：
1. **先出顺子中间的牌**：如6788910J，先出8，留下两头好调整
2. **看下家牌型**：
   - 不要单出顺子两头小张顺牌，下家要单出大单顶牌
   - 如910JQKA顺，下家不吃单，出单时套9，提
   - 下家要单，出单时顶A

### 7. 弱牌诱对手

**核心原理**：我方炸 (`Bomb`) 少，有两大夯 (`ThreeWithTwo`)，可先小顺 (`Straight`)，打出不收，遇对方封顺，对方改出夯，先弱后强，声东击西

**平台变量名**：
- `Bomb`：炸弹
- `ThreeWithTwo`：三带二（夯）
- `Straight`：顺子

### 8. 送顺先送中顺

**策略**：
1. **对家要顺 (`Straight`)，己有小顺可即送**：如能回手，先送中顺，后救小顺，防下家顶
2. **有小夯 (`ThreeWithTwo`)，不急出**：上、中场不出相克牌
3. **没小顺，等送对 (`Pair`)**：有顺就有对，对顺是一家
4. **对家仅出两夯，可能还有一大顺 (`Straight`)**：可尝试送一小顺
5. **对家三夯后可能不会有顺**
6. **顺夯都不要，尝试送小三连张**

**平台变量名**：
- `Straight`：顺子
- `ThreeWithTwo`：三带二（夯）
- `Pair`：对子

## 四、顺子的应对技巧

### 1. 顺子管到头，对手没想头

**核心原理**：出最大的顺子 (`Straight`)（如10JQKA）可以有效压制对手，逼炸 (`Bomb`) 或放弃出牌。

**平台变量名**：`Bomb`（炸弹）

### 2. 枪打头一顺

**核心原理**：当对手首次出顺子 (`Straight`) 时，应立即用更强的牌型进行压制，以阻断对手的出牌节奏。

### 3. 顺子多缺三带二，三带二多顺子无

**核心原理**：据顺子可预判对手手上的牌，出完顺子后，在顺子牌区内炸弹和三带二的概率低，反之单牌和对子的概率就高。

**判断方法**：
1. **出5——9的顺子 (`Straight`)**：大概率没有5——9的三张和炸弹 (`Bomb`)，会有单张 (`Single`) 或对子 (`Pair`)
2. **出3个J带对9**：没有7到J，10到A的顺子
3. **出777带22**：说明手上没有小顺子
4. **出大顺子如10JQKA**：则手中没有大的三带二 (`ThreeWithTwo`)
5. **出三带二 (`ThreeWithTwo`) 对手不要**：则对手手上可能有顺子 (`Straight`)
6. **先出小三带二，后用大三带二回收**：中间牌就可能是顺子

**平台变量名**：
- `Straight`：顺子
- `Bomb`：炸弹
- `Single`：单张
- `Pair`：对子
- `ThreeWithTwo`：三带二

### 4. 打顺会有对，防对你就对

**核心原理**：
- 对手打出顺子 (`Straight`)，应该考虑保留对子 (`Pair`)，以应对对手可能随后出的对牌
- 对手出了顺子后，继续出单牌或者其他牌型，可能意味着对手的手牌较弱，需要抓住机会进行攻击

### 5. 一般不接对家小顺

**核心原理**：对家先出顺 (`Straight`)，想上游，随时接，定下游，绝不要。

**策略**：
1. **牌力强**：可不接，稍后送
2. **牌力中，不想下游，有两小顺**：不影响对家，可套一小顺
3. **牌力差**：绝不接，炸对方尽快送

**平台变量名**：`Bomb`（炸弹）

### 6. 开始不要顺，后面又要顺，可能还有顺

**核心原理**：
- 我方先出顺 (`Straight`)，第三把大顺不急出，等反制对手方
- 对手方第二把所拆的中顺应不急回打，也要防我方再收顺

## 五、顺子残局运用

### 1. 尽量不留最后一手顺

**核心原理**：尽量不留最后一手顺 (`Straight`)，这也是最难有变换的牌型。

### 2. 七张八张，打顺打夯

**核心原理**：对手剩七张或者八张牌，主动打顺子 (`Straight`)。

### 3. 九张十张，不出顺夯

**核心原理**：对手剩九张或者10张牌，不要主动打顺子 (`Straight`)。

### 4. 拆牌送对家送顺

**核心原理**：
- 对家剩五张，明显是一顺（34567），本家上游无望，可拆方片同花炸 (`StraightFlush`) 23456，变A2345小顺子送对家上游（经典案例）
- 下家剩九张，一炸 (`Bomb`) 加一顺 (`Straight`)，上家送小顺，有顶出大顺，无顶炸上方，下家或起炸，后面靠对手，大炸再加炸，控对方余五张

**平台变量名**：
- `StraightFlush`：同花顺
- `Bomb`：炸弹
- `Straight`：顺子

### 5. 明牌留大王不留顺，对家好送牌

**核心原理**：己剩两手牌，一王 (`R`) 加中顺 (`Straight`)。顺、王都为大，先出顺，留明王，对家好送牌。

**平台变量名**：
- `R`：大王
- `Straight`：顺子

### 6. 小王不大留中顺

**核心原理**：己剩两手牌，小王 (`B`) 加中顺 (`Straight`)：
1. **对方有小顺 (`Straight`)，出王逼炸 (`Bomb`)**：留顺待压
2. **对方能接顺，出王逼炸**：留顺解放对家
3. **上家剩一大王 (`R`)，对家有小顺及大炸 (`Bomb`)**：冲出小王，待对家炸大王救己中顺

**平台变量名**：
- `B`：小王
- `Straight`：顺子
- `Bomb`：炸弹
- `R`：大王

### 7. 接对家报听后小顺

**核心原理**：
- 对家炸 (`Bomb`) 后倒数第二手，放一小顺 (`Straight`)，明显剩炸，本家有顺要抢接
- 本家首打顺，曾被对方封，对家炸后或中下场发小顺要接顺

**平台变量名**：
- `Bomb`：炸弹
- `Straight`：顺子

## 注意事项

1. **顺子的5、10法则**：所有的小顺子都必定包含5，所有的大顺子都必定包含10
2. **顺子封到顶**：顺子最大也就费个A
3. **组顺生两单，肯定没眼光**：顺子的组牌内单牌不应超过两张
4. **谁打谁收**：先顺后夯还有顺，先夯后顺还有夯
5. **小顺往前凑，大顺必殿后**：牌中有多个顺子，先出较小的顺子
