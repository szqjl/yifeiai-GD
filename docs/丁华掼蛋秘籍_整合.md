# 丁华《掼蛋技巧秘籍》 — OCR 整合摘要

来源: `docs/skill/掼蛋技巧秘籍_OCR.txt`（OCR 提取；约 160 页，已载入工作区）

目的: 将 OCR 文本中的关键规则、策略、配合与实战技巧提取为结构化要点，供知识注入系统、规则型 AI 和训练流水线使用。

快速概览要点：
- 规则类要点：明确出牌优先级、主牌/副牌使用时机、炸弹与拆牌原则。
- 配合策略：强调队友间的让牌与配合节奏、利用队友信息形成“升级/封堵”联动。
- 观察与判断：记牌与读牌（通过牌型、出牌顺序估算对手可能持牌），并据此调整拆牌/保牌策略。
- 升级与防守：在关键回合保留阻止对方升级的牌（或保留炸弹以阻挡对手），并在合适时机触发升级策略。
- 时机掌握：首轮进攻与留牌权衡、何时用炸弹、何时保存主牌以做终结。

建议用于工程化处理的工作流：
1) 将原始 OCR 文本作为 `source` 导入 `KnowledgeLearningPipeline`；对文本做轻量清洗（去重、合并换行、常见 OCR 纠错）。
2) 使用 `RealExpertKnowledgeExtractor`（或 `ExpertKnowledgeExtractor`）按类别抽取知识项（rule/strategy/tip/tactic），并将抽取结果格式化为 `StandardGuandanKnowledge`。
3) 把高置信度项（confidence >= 0.8）优先注入 `KnowledgeInjectionSystem` 并存入数据库/记忆层；低置信度项标注为待人工校验。
4) 对抽取出的典型规则生成对应的可执行策略（YAML/JSON），并纳入 `GuandanKnowledgeBase.build_strategy_rules()` 的输入。
5) 为训练流水线（self-play）生成基准化的示例对局（把常见情形与建议操作转换为 replay case），用于评估规则有效性。

建议的紧急/优先级划分（示例）：
- 优先级高：直接影响胜负或阻止对方升级的规则（保炸弹、阻止升级、主要防守规则）。
- 优先级中：队友配合与升级节奏相关策略。
- 优先级低：技巧性提示、示例性对局或可选风格性策略。
